What is Backbone Marionette and why would you use it?

  • Composite application library for Backbone.js
  • Collection of common design and implementation patterns
  • Brings an application architecture to Backbone
  • View management and memory management
  • Reduce boilerplate code

Single page apps

  • Serve a single HTML page
  • Bootstrap as much data as you need to start the application (embedded in the page as JSON)
  • Ajax requests to pick up and post data back as needed

Dependencies

  • Backbone.js
  • Underscore.js
  • Jquery (for deferred events) or Zepto (untested)

Regions

Regions can be defined on the app, or as part of a layout

ItemView

Used to render a model

Serialize data vs model

CompositeView

Used to render collections (e.g. lists, tables etc)

Layout

Used in place of an ItemView when you want to render more than one model or collection in different regions

Events

App level event aggregator used to broadcast and listen to events in separate parts of the app, allowing it to be built in a loosely coupled way

Router

Reduce the boilerplate code of handling route events and then calling a single method on another object. Have your routers configured to call the method on your object, directly.

Modules

useful for creating modular, encapsulated applications
